Here's the full startup sequence:

1. Prerequisites
Install these if you don't have them:

Python 3.11+
Node.js 18+
Redis (for Celery task queue)
On Windows, Redis is easiest via Docker:


docker run -d -p 6379:6379 redis:alpine
2. First-time setup

# Clone env and fill in your keys
cp .env.example .env
# Edit .env: set DATABASE_URL, ANTHROPIC_API_KEY at minimum

# Install all deps
make install
# Or manually:
cd backend; pip install -e ".[dev]"
cd ..\frontend; npm install
3. Initialize the database

cd backend
python ..\scripts\setup_db.py    # creates tables
python ..\scripts\seed_stocks.py  # seeds ~30 Tier 1 symbols
4. Run the dev servers
Option A — two terminals (recommended on Windows):


# Terminal 1
cd backend
uvicorn main:app --reload --port 8000

# Terminal 2
cd frontend
npm run dev
Option B — Unix/WSL/Git Bash:


make dev
Then open:

Frontend: http://localhost:3000
API docs: http://localhost:8000/docs
5. Generate your first suggestions

make agents
# or:
cd backend && python -m agents.orchestrator
This runs the full LangGraph pipeline — Scanner → TA + Sentiment + Pattern → Claude synthesizer. Results appear in the /api/suggestions/ endpoint and on the Suggestions page.

6. Run EOD data ingest

make ingest
# or:
cd backend && python -m data.ingest
Normally this runs automatically via Celery Beat at 9:30 PM UTC on weekdays. To start the scheduler manually:


cd backend
celery -A scheduler.tasks.celery_app beat --loglevel=info &
celery -A scheduler.tasks.celery_app worker --loglevel=info
7. Run tests

make test
# or just:
pytest
The risk tests in tests/test_risk.py run without any external services. The agent and API tests need the backend importable (handled by pytest.ini).

Minimum viable run (no Redis, no Telegram): fill in DATABASE_URL + ANTHROPIC_API_KEY, run setup_db.py, then uvicorn + npm run dev + make agents. Everything else is optional for local testing.

Step 1 — Run today's data ingest
Open a terminal in D:\swing-trading-system\backend and run:

cd D:\swing-trading-system\backend
python -m scheduler.ingest_job
This fetches the last 30 days of OHLCV prices from yfinance for all 147 stocks and upserts them into daily_prices. It takes ~30–60 seconds. You'll see output like:

IngestResult(2026-05-15): 147 upserted, 0 skipped, 0 missing — 42.3s
Step 2 — Run the AI agents to generate today's signals
cd D:\swing-trading-system\backend
python -m agents.orchestrator
This runs the full LangGraph pipeline — scanner → TA + Sentiment + Pattern agents → LLM synthesizer — and writes Suggestion rows into the database. Takes 1–3 minutes depending on Claude API latency. After this, the /suggestions page will show cards.

Data persistence
Data is stored in Supabase (PostgreSQL) — it persists permanently. Once you've run the ingest and agents once, those rows stay in the database. Every subsequent frontend load reads from the DB, so you don't need to re-run anything unless you want fresh data for a new trading day
